## 1. Executive Summary

GD(지드래곤) 음성을 복제한 TTS 모델 파인튜닝 프로젝트의 종합 보고서입니다.
v3부터 v10까지 총 8개 버전의 실험을 통해 최적의 모델을 탐색하였습니다.

v5 모델이 최종 선택되어 실시간 음성통화 서비스에 적용되었습니다.

## 6. 추론 성능 분석

### 6.1 추론 속도 측정

실제 서비스 로그 기반 성능 분석:

| 텍스트 길이 | 오디오 길이 | 처리 시간 | RTF |
|-------------|-------------|-----------|-----|
| 10자 | 1.2s | 871ms | 0.73 |
| 14자 | 2.6s | 1,777ms | 0.68 |
| 28자 | 3.7s | 2,204ms | 0.60 |
| 39자 | 4.6s | 2,991ms | 0.65 |
| 77자 | 8.9s | 5,505ms | 0.62 |

**평균 RTF**: 0.65 (실시간 대비 35% 빠름)

> RTF (Real-Time Factor) = 처리 시간 / 오디오 길이
> RTF < 1.0: 실시간 스트리밍 가능

### 6.2 메모리 사용량

| 항목 | 사용량 |
|------|--------|
| 모델 로드 (파인튜닝) | 3.89 GB |
| 모델 로드 (클론) | 추가 ~3.5 GB |
| 추론 시 피크 | ~5.5 GB |
| 권장 GPU VRAM | 8 GB 이상 |

### 6.3 응답 지연 (Latency)

실시간 음성통화 파이프라인 성능:

```
[STT → LLM → TTS 파이프라인]

STT (Whisper large-v3): 135~207ms
LLM (Gemini 3 Flash): 1,333~2,321ms (TTFA)
TTS (GD v5): 1,000~3,500ms (첫 청크)

총 TTFA: 3.5~6.0초
```

| 단계 | 평균 지연 |
|------|-----------|
| 첫 응답까지 (TTFA) | ~4초 |
| 전체 파이프라인 | ~10초 |

### 6.4 스트리밍 성능

- **청크 크기**: 4,800 bytes (100ms @ 24kHz)
- **첫 오디오 청크**: TTS 시작 후 ~1초
- **연속 스트리밍**: 지연 없이 전달

1. **MFCC 유사도 (음색)**: 모든 샘플에서 **98~99%** 이상
   - 파인튜닝이 GD의 음색을 매우 정확히 학습함
   - 베이스라인 클론과 큰 차이 없음

2. **Pitch 유사도 (음높이)**: 샘플별 **62~97%** 변동
   - 긴 문장에서 더 높은 유사도 (안정화)
   - 짧은 문장에서 변동성 있음

3. **파인튜닝 vs 클론 비교**:
   ```
   파인튜닝 (v5) 평균 유사도: 93.9%
   베이스라인 (clone) 평균 유사도: 91.7%

   개선율: +2.2%p
   ```

### 8.6 모드별 유사도 비교

| 모드 | 평균 유사도 | 장점 | 단점 |
|------|-------------|------|------|
| **Finetuned (v5)** | **93.9%** | 음색 안정성, 자연스러운 억양 | 재훈련 필요 |
| Clone (ICL) | 91.7% | 즉시 적용 가능 | pitch 변동성 |

### 8.7 결론

✅ **유사도 목표 달성**: 평균 93.9% 유사도로 GD 음성 복제 성공

**세부 평가:**
- **음색 (MFCC)**: ⭐⭐⭐⭐⭐ 99% - 거의 완벽
- **음높이 (Pitch)**: ⭐⭐⭐⭐ 85~97% - 문장 길이에 따라 변동
- **음색 밝기 (Spectral)**: ⭐⭐⭐⭐ 85~96% - 양호

**권장사항:**
- 긴 문장(10자 이상)에서 더 안정적인 결과
- 짧은 문장은 문맥 추가로 개선 가능
- 특정 억양 패턴 추가 훈련으로 pitch 유사도 향상 가능

## 9. 결론 및 권장사항

### 9.1 프로젝트 성과

✅ **목표 달성**
- GD 음성 복제 모델 성공적 훈련
- 실시간 음성통화 서비스 배포 완료
- RTF 0.65로 실시간 스트리밍 가능
- 8개 버전 실험을 통한 최적 설정 도출

### 9.2 핵심 학습

| 순위 | 학습 내용 | 영향도 |
|------|----------|--------|
| 1 | **데이터 품질 > 양**: 18.8분의 깨끗한 음성이 효과적 | 높음 |
| 2 | **증강 주의**: 음색 보존이 중요하면 원본만 사용 | 높음 |
| 3 | **Loss ≠ 품질**: 낮은 Loss가 좋은 음성이 아님 | 높음 |
| 4 | **에폭 최적화**: 100 에폭 이후 수렴, 150이 적정선 | 중간 |
| 5 | **LR 5e-7**: 음성 도메인에서 안정적 | 중간 |

### 9.3 최적 훈련 설정 (권장)

```yaml
추천_설정:
  모델: Qwen3-TTS-12Hz-1.7B-Base
  Learning_Rate: 5e-7
  Epochs: 150
  Batch_Size: 1 (VRAM 제약 시)
  LR_Scheduler: Cosine
  데이터_증강: 없음 (원본만)

검증_기준:
  - Loss < 10.0
  - 청취 테스트 통과
  - 새로운 문장에서 자연스러움
```

### 9.4 향후 개선 방향

| 우선순위 | 개선 항목 | 예상 효과 |
|----------|-----------|-----------|
| 1 | 더 많은 고품질 음성 수집 | 다양한 감정/톤 표현 |
| 2 | 감정 제어 파라미터 추가 | 상황별 음성 변조 |
| 3 | 실시간 음성 변환 (RVC) | 더 정확한 음색 복제 |
| 4 | 다국어 지원 | 영어/일본어 발화 |

### 9.5 운영 권장사항

```yaml
프로덕션_설정:
  모델: gd-voice-v5/checkpoint-epoch-final
  모드: finetuned (기본)
  백업: clone (ICL)

모니터링:
  - TTS 처리 시간 < 3초 (10자 기준)
  - GPU 메모리 < 6GB
  - 에러율 < 1%

스케일링:
  - 동시 요청: GPU당 1개 (직렬 처리)
  - 수평 확장: 다중 GPU 또는 인스턴스
```
