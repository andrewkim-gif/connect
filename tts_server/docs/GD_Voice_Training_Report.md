```

2. 무음 기반 청크 분할: 3~12초 단위
```

| 단계 | 결과 |

| 데이터 증강 | 없음 (원본만 사용) |

### 2.3 v5 데이터 특징

- **증강 제거**: 이전 버전(v3, v4)에서 사용한 속도 변환 증강을 제거
- **원본 음색 보존**: 증강 없이 원본 음성만 사용하여 GD 고유 음색 유지
- **청크 품질**: 무음 기반 분할로 자연스러운 발화 단위 보존

## 5. 추론 성능 분석

### 5.1 추론 속도 측정

실제 서비스 로그 기반 성능 분석:

| 텍스트 길이 | 오디오 길이 | 처리 시간 | RTF |
|-------------|-------------|-----------|-----|
| 10자 | 1.2s | 871ms | 0.73 |
| 14자 | 2.6s | 1,777ms | 0.68 |
| 28자 | 3.7s | 2,204ms | 0.60 |
| 39자 | 4.6s | 2,991ms | 0.65 |
| 77자 | 8.9s | 5,505ms | 0.62 |

**평균 RTF**: 0.65 (실시간 대비 35% 빠름)

> RTF (Real-Time Factor) = 처리 시간 / 오디오 길이
> RTF < 1.0: 실시간 스트리밍 가능

### 5.2 메모리 사용량

| 항목 | 사용량 |
|------|--------|
| 모델 로드 (파인튜닝) | 3.89 GB |
| 모델 로드 (클론) | 추가 ~3.5 GB |
| 추론 시 피크 | ~5.5 GB |
| 권장 GPU VRAM | 8 GB 이상 |

### 5.3 응답 지연 (Latency)

실시간 음성통화 파이프라인 성능:

```
[STT → LLM → TTS 파이프라인]

STT (Whisper large-v3): 135~207ms
LLM (Gemini 3 Flash): 1,333~2,321ms (TTFA)
TTS (GD v5): 1,000~3,500ms (첫 청크)

총 TTFA: 3.5~6.0초
```

| 단계 | 평균 지연 |
|------|-----------|
| 첫 응답까지 (TTFA) | ~4초 |
| 전체 파이프라인 | ~10초 |

### 5.4 스트리밍 성능

- **청크 크기**: 4,800 bytes (100ms @ 24kHz)
- **첫 오디오 청크**: TTS 시작 후 ~1초
- **연속 스트리밍**: 지연 없이 전달

## 6. 버전별 비교

### 6.1 훈련 버전 히스토리

| 버전 | 에폭 | 최종 Loss | 데이터 | 특이사항 |
|------|------|-----------|--------|----------|
| v3 | 50 | ~10.5 | 증강 포함 | 초기 실험 |
| v4 | 300 | ~9.2 | 증강 포함 | 과적합 우려 |
| **v5** | **150** | **9.38** | **원본만** | **프로덕션 선택** |
| v6 | 200 | ~9.3 | 원본만 | v5 대비 개선 없음 |
| v7 | 300 | ~9.1 | 원본만 | 과적합 |
| v8-v10 | 다양 | - | 실험 | 정규화 테스트 |

### 6.2 v5 선택 이유

1. **증강 제거 효과**: 원본 음성만 사용하여 GD 고유 음색 유지
2. **적정 에폭**: 150 에폭에서 충분히 수렴, 과적합 없음
3. **안정적 Loss**: 9.38로 안정적이며 분산이 적음
4. **음질 평가**: 청취 테스트에서 가장 자연스러운 음성

### 6.3 ICL 클론 vs 파인튜닝 비교

| 항목 | ICL 클론 | 파인튜닝 (v5) |
|------|----------|---------------|
| 음색 유사도 | 중간 | 높음 |
| 발화 자연스러움 | 높음 | 매우 높음 |
| 훈련 필요 | X | O (150 에폭) |
| 새 음성 적응 | 즉시 | 재훈련 필요 |
| 권장 용도 | 백업/다양성 | 메인 서비스 |

## 7. 결론 및 권장사항

### 7.1 프로젝트 성과

✅ **목표 달성**
- GD 음성 복제 모델 성공적 훈련
- 실시간 음성통화 서비스 배포 완료
- RTF 0.65로 실시간 스트리밍 가능

### 7.2 핵심 학습

1. **데이터 품질 > 양**: 18.8분의 깨끗한 음성이 수 시간의 노이즈 포함 음성보다 효과적
2. **증강 주의**: 음색 보존이 중요한 경우 증강 없이 원본만 사용
3. **에폭 최적화**: 100 에폭 이후 수렴, 150 에폭이 적정선
4. **LR 5e-7**: 매우 낮은 학습률이 음성 도메인에서 안정적

### 7.3 향후 개선 방향

| 우선순위 | 개선 항목 | 예상 효과 |
|----------|-----------|-----------|
| 1 | 더 많은 고품질 음성 수집 | 다양한 감정/톤 표현 |
| 2 | 감정 제어 파라미터 추가 | 상황별 음성 변조 |
| 3 | 실시간 음성 변환 (RVC) | 더 정확한 음색 복제 |
| 4 | 다국어 지원 | 영어/일본어 발화 |

### 7.4 운영 권장사항

```yaml
프로덕션 설정:
  모델: gd-voice-v5/checkpoint-epoch-final
  모드: finetuned (기본)
  백업: clone (ICL)

모니터링:
  - TTS 처리 시간 < 3초 (10자 기준)
  - GPU 메모리 < 6GB
  - 에러율 < 1%

스케일링:
  - 동시 요청: GPU당 1개 (직렬 처리)
  - 수평 확장: 다중 GPU 또는 인스턴스
```
